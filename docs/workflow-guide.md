## Создание своей рабочей области

### Шаг 1: Начало работы с курсом
1. Найдите нужный курс в папке `courses/`
2. Изучите папку `assignments/` - там все задания
3. Прочитайте README.md в папке задания

### Шаг 2: Создание личной папки
**Правильное именование:**
courses/название-курса/assignments/номер-задания/фамилия-имя/

**Примеры:**
✅ ПРАВИЛЬНО:
courses/ai-business-models/assignments/assignment-1/ivanov-ivan/

❌ НЕПРАВИЛЬНО:
courses/ai-business-models/assignments/assignment-1/ivan/ (только имя)
courses/ai-business-models/assignments/assignment-1/Иван Иванов/ (кириллица)
courses/ai-business-models/assignment-1/ivanov-ivan/ (неправильный путь)

### Процесс работы
1. **Регулярно коммитьте изменения:**
# После каждого значимого этапа
git add courses/your-course/assignments/assignment-1/your-name/
git commit -m "Добавил анализ конкурентов"
git push

# Используйте понятные сообщения коммитов:
✅ ХОРОШО:
"Добавил раздел методологии исследования"
"Исправил ошибки в расчетах"
"Добавил графики и визуализации"

❌ ПЛОХО:
"исправления"
"123"
"коммит"

## Сдача задания
# Шаг 1: Финальная проверка
Перед сдачей убедитесь, что:
- Все файлы находятся в вашей личной папке
- README.md заполнен и описывает работу
- Файлы названы понятно
- Работа соответствует требованиям задания

# Шаг 2: Финальный коммит
# Добавляем все изменения
git add courses/your-course/assignments/assignment-1/your-name/

# Создаем финальный коммит
git commit -m "Задание 1: финальная версия. Выполнены все требования"

# Отправляем на GitHub
git push

# Шаг 3: Уведомление преподавателя
После отправки работы:
- Перейдите на вкладку "Issues" в репозитории
- Найдите Issue с вашим именем и заданием
- Переместите карточку в колонку "Under Review"
- Напишите комментарий:
@TeacherUsername Задание 1 готово к проверке.
Ссылка на работу: [ваша папка]

## Получение обратной связи
# Где искать комментарии преподавателя:
1) В Issue - общие замечания и оценка
2) В комментариях к коммитам - конкретные замечания по коду
3) В самом коде - line comments в Pull Request (если используется)
4) На доске Projects - статус выполнения

# Ответ на замечания:
Если нужно исправить работу после проверки:
- Внесите исправления в локальные файлы
- Сделайте коммит:
git add .
git commit -m "Исправления по замечаниям: переработал выводы, добавил источники"
git push
- Напишите в Issue: "Исправления внесены, готово к повторной проверке"

### Важные правила
- Работайте только в своей папке
- Не изменяйте чужие файлы
- Соблюдайте сроки сдачи
- Следуйте требованиям к оформлению
- Задавайте вопросы в Issues если что-то непонятно
